## 1. 产品概述

### 1.1 产品定位

Holdem 是一款基于微信小程序的社交德州扑克游戏，主打与好友一起畅玩的轻量级娱乐体验。

### 1.2 目标用户

- 德州扑克爱好者
- 希望与朋友进行休闲娱乐的微信用户
- 想要学习德州扑克规则的新手玩家
- 暂时不考虑上架小程序，只有测试环境在内部用户里使用

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 社交性 | 支持多人在线对战，与好友同乐 |
| 便捷性 | 微信小程序无需下载，即开即玩 |
| 公平性 | 纯筹码系统，无充值，每日签到获取筹码 |

### 1.4 产品形态

- **平台**: 微信小程序
- **设备**: 移动端（iOS/Android）
- **屏幕方向**: 大厅页面竖屏，游戏页面横屏

## 3. 页面详细设计

### 3.1 登录页 (LoginPage)

**页面描述**: 用户首次进入应用或登录态失效时展示的授权登录页面。

**UI 布局**:
- 页面中央：应用 Logo（黑桃 + 红心图标）
- 应用名称：德州扑克
- 副标题：与好友一起畅玩德州
- 登录按钮：微信授权登录（绿色主按钮）
- 页面底部：用户协议链接

**功能说明**:

| 功能项 | 说明 |
|--------|------|
| 微信授权登录 | 调用微信 API 获取用户头像、昵称 |
| 用户协议 | 点击跳转用户协议页面 |
| 登录成功 | 跳转至大厅页面 |

**业务规则**:
- 新用户首次登录自动注册，初始筹码 10,000
- 登录态有效期 7 天

**UI 布局**:

- 搜索栏：搜索房间号/房间名
- 房间列表：可滚动的房间卡片列表
- 底部导航栏：大厅 | 我的

**房间卡片信息**:

| 房间名称 | 房主自定义 | 房间 #1234 |
| 盲注信息 | 大盲注金额 | 大盲注: 50 |
| 人数状态 | 当前/最大人数 | 4/8 |
| 加入按钮 | 点击加入房间 | 加入 |

**功能说明**:

| 功能项 | 说明 |
|--------|------|
| 搜索房间 | 支持按房间号、房间名模糊搜索 |
| 创建房间 | 弹出创建房间弹窗 |
| 加入房间 | 弹出加入房间弹窗确认买入筹码 |

### 3.3 创建房间弹窗 (CreateRoomDialog)

**UI 布局**:
- 标题：创建房间
- 房间名称输入框
- 大盲注金额输入框
- 带入筹码选择：1000 | 2000 | 5000 | 自定义金额
- 操作按钮：取消 | 创建房间

**业务规则**:

| 规则 | 说明 |
|------|------|
| 房间名称 | 1-20 字符，支持中英文数字 |
| 大盲注 | 最小 10，最大 1000 |
| 小盲注 | 自动设为大盲注的 1/2 不可修改 |
| 带入筹码 | 房主本次带入的筹码，必须 ≥ 大盲注 × 20 且 ≤ 玩家当前筹码 |
| 房间有效期 | 所有玩家离开后自动销毁 |

### 3.4 加入房间弹窗 (JoinRoomDialog)

**UI 布局**:
- 标题：加入房间
- 房间信息：房间名 | 大盲注
- 带入筹码选择：1000 | 2000 | 5000 | 自定义金额
- 操作按钮：取消 | 加入房间

**业务规则**:
- 带入筹码必须 ≥ 大盲注 × 20 且 ≤ 玩家当前筹码
- 加入成功后跳转至游戏页面

**顶部信息栏**:
- 返回/退出按钮

**座位信息**:

| 元素 | 说明 |
|------|------|
| 头像 | 玩家微信头像/默认头像 |
| 昵称 | 玩家昵称 |
| 筹码 | 当前持有筹码 |
| 状态标识 | 说话中/已弃牌/全下 |
| 下注筹码 | 当前轮下注金额 |

| 状态 | 显示内容 |

### 3.6 玩家状态流转

```
┌─────────────┐
│   进入房间   │
└──────┬──────┘
       ▼
┌─────────────┐     选择座位      ┌─────────────┐
│  站立观战   │ ───────────────▶ │  入座等待   │
└─────────────┘                   └──────┬──────┘
       ▲                                 │
       │         离席                    │ 游戏开始
       │◀────────────────────────────────┤
       │                                 ▼
       │                          ┌─────────────┐
       │         弃牌/游戏结束     │  游戏中     │
       │◀────────────────────────│             │
       │                          └─────────────┘
       │
       ▼
┌─────────────┐
│  退出房间   │

```

**状态说明**:

| 状态 | 描述 | 可执行操作 |
|------|------|-----------|
| 站立观战 | 进入房间默认状态 | 选座入座、退出房间 |
| 入座观战 | 游戏进行中入座 | 离席、等待下局 |
| 入座等待 | 等待游戏开始 | 离席、准备参与 |
| 游戏中 | 正在参与游戏 | 弃牌、过牌、跟注、加注、全下 |

### 3.7 游戏流程

#### 3.7.1 开局流程

```
1. 房主点击「开始游戏」
2. 所有入座玩家选择「参与」或「不参与」
3. 30 秒内未选择自动视为「不参与」
4. 至少 2 人参与，游戏开始
5. 系统自动发牌，确定庄家位置
```

#### 3.7.2 游戏阶段

| 阶段 | 英文 | 公共牌 | 说明 |
|------|------|--------|------|
| 翻牌前 | Preflop | 0 张 | 发放 2 张底牌 |
| 翻牌 | Flop | 3 张 | 翻开 3 张公共牌 |
| 转牌 | Turn | 4 张 | 翻开第 4 张公共牌 |
| 河牌 | River | 5 张 | 翻开第 5 张公共牌 |
| 摊牌 | Showdown | 5 张 | 比较牌型大小 |

#### 3.7.3 结算流程

```
1. 游戏结束，展示胜者及牌型
2. 底池筹码分配给胜者
3. 弹出「下一局」确认框
   - 参与：继续参与下一局
   - 不参与：以观战状态留在座位
4. 所有玩家确认后开始下一局
```

## 5. 接口设计

### 5.1 认证接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/wechat-login` | POST | 微信授权登录 |
| `/api/auth/me` | GET | 获取当前用户信息 |

### 5.2 房间接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/rooms` | GET | 获取房间列表 |
| `/api/rooms` | POST | 创建房间 |
| `/api/rooms/:id` | GET | 获取房间详情 |
| `/api/rooms/:id/join` | POST | 加入房间 |
| `/api/rooms/:id/leave` | POST | 离开房间 |
| `/api/rooms/:id/sit` | POST | 入座 |
| `/api/rooms/:id/stand` | POST | 站起 |

### 5.3 游戏接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/game/:roomId/start` | POST | 开始游戏 |
| `/api/game/:roomId/action` | POST | 执行操作 |
| `/api/game/:roomId/ready` | POST | 准备下一局 |

### 5.4 用户接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/users/:id/stats` | GET | 获取用户统计 |
| `/api/users/:id/history` | GET | 获取游戏历史 |
| `/api/users/checkin` | POST | 每日签到 |

### 5.5 WebSocket 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `room:join` | C→S | 加入房间 |
| `room:leave` | C→S | 离开房间 |
| `room:update` | S→C | 房间状态更新 |
| `game:start` | S→C | 游戏开始 |
| `game:action` | C→S | 玩家操作 |
| `game:turn` | S→C | 轮到某玩家 |
| `game:cards` | S→C | 发牌/翻牌 |
| `game:end` | S→C | 游戏结束 |
| `player:join` | S→C | 玩家加入 |
| `player:leave` | S→C | 玩家离开 |
| `player:sit` | S→C | 玩家入座 |
| `player:stand` | S→C | 玩家站起 |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 2 秒 |
| 操作响应时间 | ≤ 300ms |
| WebSocket 延迟 | ≤ 200ms |
| 并发房间数 | ≥ 10 |
| 单房间玩家数 | 8 人 |

### 6.2 安全要求

- 用户数据加密存储
- 通信使用 HTTPS/WSS
- 防止筹码作弊
- 操作频率限制

### 6.3 兼容性

- 微信版本 ≥ 7.0.0
- iOS ≥ 11.0
  - iPhone 14 Pro 及以上
- Android ≥ 6.0
